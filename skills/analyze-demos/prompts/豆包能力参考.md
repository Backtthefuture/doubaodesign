# 豆包模型能力参考知识库

本文档作为"能力跃迁"分析的参考知识库，包含豆包 1.6 和 1.8 的核心能力差异。

---

## 豆包 1.6 核心能力

### 基础能力
- **256K 超长上下文**：最大输入 224K tokens，输出最大 16K tokens
- **4档思考长度**：Minimal / Low / Medium / High 四档可调
- **多模态输入**：支持文字、图片、视频输入
- **单次视频理解**：640帧/次

### 特色功能
- **GUI 操作**：基于视觉的浏览器交互能力
- **边想边搜**：基于缺失信息自动搜索
- **DeepResearch**：快速生成调研报告
- **三种思考模式**：auto / thinking / non-thinking

### 评测成绩
- GPQA Diamond：81.5分
- AIME25 数学测评：86.3分
- 高考数学全国新一卷：144分

---

## 豆包 1.8 核心能力

### 架构升级
- **原生多模态**：从预训练阶段就端到端训练，不再是 LLM + VLM 组合
- **视觉理解质变**：性能仅次于 Gemini 3 Pro

### Agent 能力突破
- **OS Agent**：可操作电脑、网页、移动端三类环境
- **跨应用自动化**：无需 API 也能操作系统级任务
- **思考+工具一体化**：思考模式下保留推理状态进行多次工具调用

### 视频理解升级
- **帧数翻倍**：单次视频理解从 640帧 → 1280帧
- **超长视频支持**：低帧率理解 + 关键帧高精度分析
- **应用场景**：在线教育课程、工业质检视频

### 其他增强
- **复杂指令遵循**：大幅增强
- **工具调用稳定性**：复杂自定义工具格式输出稳定，报错率极低
- **多步任务规划**：规划与执行水平大幅提升

### 评测成绩
- BrowseComp-en：67.6分（全球领先，超过 Gemini 3 Pro）
- ZeroBench 视觉推理：11.0分（最高分）
- VLMsAreBiased 视觉问答：62.0分

## 典型场景能力对比

### 场景1：视频处理类
- **1.6**：640帧限制，约21秒30fps视频，超长视频需分段
- **1.8**：1280帧 + 关键帧分析，可理解2小时完整视频

### 场景2：自动化类
- **1.6**：仅浏览器 GUI 操作，需要页面可见
- **1.8**：OS Agent，可操作桌面应用、跨应用数据搬运

### 场景3：复杂代码生成
- **1.6**：可生成，但长代码偶有格式问题
- **1.8**：工具调用稳定性极高，复杂代码输出更规范

### 场景4：多步骤任务
- **1.6**：需要明确分步指令
- **1.8**：自主规划多步骤，思考与执行融合
