# 豆包创意内容完整生成

根据输入的 Demo Prompt 描述，一次性生成完整的豆包创意内容。**直接输出JSON，不要任何解释。**

---

## 核心原则

**基于Demo的实际内容，生成准确且有吸引力的创意描述。**

- 忠于原Demo的核心功能，不要生造不相关的特性
- 用更生动、专业的语言重新包装，提升表达质量
- 根据Demo类型自然地突出其亮点

---

## 高分参考（按需采用，不强制）

以下是不同类型Demo的加分元素，**仅在与Demo内容相关时采用**：

### 趣味分加分元素
| 类型 | 适用场景 | 可考虑的表达 |
|------|----------|--------------|
| 3D/可视化 | Demo涉及3D、图形渲染 | "3D"、"可视化"、"沉浸式"、"实时渲染" |

| 类型 | 适用场景 | 可考虑的表达 |
|------|----------|--------------|

## 输出格式（必须严格遵守）

```json
{
  "title": "6-12字标题",
  "description": "40-80字描述",
  "tags": "标签1, 标签2, 标签3",
  "example_prompt": "20-50字示例Prompt",
  "steps": [
    "用户说：...",
    "1.AI...",
    "2.AI...",
    "3.AI...",
    "4.AI...",
    "5.AI...",
    "6.AI...",
    "7.AI..."
  ]
}
```

**steps字段格式要求（极其重要）**：
- 第一项：用户需求，以"用户说：..."开头
- 第2-8项：必须以"数字.AI"开头，如"1.AI..."、"2.AI..."
- 每步包含具体技术细节（框架/库/方法）

---

## 字段规范

### 1. title（豆包创意标题）
- **长度**: 6-12字
- **原则**:
  - 准确概括Demo核心功能
  - 用词专业有吸引力
  - 避免过于平淡（如"XX工具"、"XX助手"可以更生动）

**好标题示例**：
| Demo类型 | 一般标题 | 更好的标题 |
|----------|----------|------------|
| 3D积木编辑 | 积木搭建工具 | 3D乐高创意搭建系统 |
| 台球游戏 | 台球游戏 | 真实物理台球模拟器 |
| 文档分析 | 文档分析工具 | 智能文档深度解析 |
| 视频理解 | 视频分析 | 长视频智能摘要生成 |

### 2. description（豆包创意描述）
- **长度**: 40-80字
- **结构建议**:
  1. 开头说明利用的核心能力
  2. 中间描述主要功能和特点
  3. 结尾点明价值或效果

**描述模板**：
```
利用豆包[核心能力]，[实现什么功能]。支持[特点1]、[特点2]、[特点3]，[价值/效果说明]。
```

### 3. tags（豆包标签）
- **数量**: 2-3个标签，逗号分隔
- **原则**: 反映Demo的核心特征，不要生造

### 4. example_prompt（示例Prompt）
- **长度**: 20-50字
- **风格**: 口语化自然表达，像真实用户说话
- **格式**: 以"帮我..."、"我想..."开头

### 5. steps（实现步骤）
- **数量**: 7-8步，JSON数组格式
- **要素**: 每步包含具体技术细节
- **第一步**: 用户输入需求（以"用户说：..."开头）

## 避免的问题

### ❌ 不要这样做
- 为了加分强行添加"3D"、"沉浸式"等与Demo无关的词汇
- 为文本处理类Demo加上"效率提升10倍"等夸大数字
- 标题与实际功能不符
- 描述中堆砌与Demo无关的特性

### ✅ 应该这样做
- 准确理解Demo的核心功能
- 用专业、生动的语言重新表达
- 只在相关时使用加分词汇
- 保持内容真实可信

---

## 重要提醒

1. **直接输出JSON**，不要任何解释文字
2. **所有字段必填**，不要遗漏
3. **steps必须是数组**，不是字符串
4. **steps格式必须严格遵守**：
   - 第一项以"用户说：..."开头
   - 后续每项以"数字.AI"开头（1.AI、2.AI、3.AI...）
   - 每步必须包含具体技术细节
5. **忠于原Demo**：基于实际内容生成，不要生造无关特性
