# 豆包创意评分标准提示词

## 任务
根据"豆包创意标题"和"豆包创意描述"，给出趣味分和实用分两个评分（各50分）。

## 输入说明
你将收到两个字段：
- **豆包创意标题**：6-12字的创意名称
- **豆包创意描述**：40-80字的功能描述

请基于这两个字段内容进行评分。

## 重要原则
1. **趣味分和实用分是独立维度**，高趣味不代表低实用，反之亦然
2. **严格按类型加减分**，不要遗漏加分项
3. **3D/游戏/编辑器类趣味分通常在30-42之间**，除非确实平淡
4. **总分 = 趣味分 + 实用分，满分100分**

## 评分维度

### 趣味分 (1-50)
衡量创意的娱乐性、创意性、视觉冲击力

| 分数区间 | 标准 |
|----------|------|
| 40-50 | 极具创意，视觉震撼，让人惊叹"这也能做到？" |
| 30-39 | 有趣味，展示效果好，能引发兴趣 |
| 20-29 | 一般创意，中规中矩 |
| 10-19 | 缺乏新意，较为平淡 |
| 1-9 | 几乎无趣味性 |

**加分项（必须应用！）**：
- 标题/描述涉及3D可视化/3D场景 → 基础分30+，再加5-10分
- 标题/描述涉及游戏类 → 基础分32+，再加5-8分
- 标题/描述涉及编辑器/创作工具 → 基础分28+，再加5分
- 标题/描述涉及视觉特效类 → 基础分30+，再加8分
- 标题/描述涉及操作系统模拟 → 基础分35+
- 描述中体现视觉效果出众 +5分
- 标题新颖独特、有创意感 +5分
- "第一次见到这种用法" +8分

**减分项**：
- 描述偏向纯文本输出 -5分
- 描述中功能常见重复 -5分
- 整体效果平淡无亮点 -3分

### 实用分 (1-50)
衡量创意的实际应用价值、解决真实问题的能力

| 分数区间 | 标准 |
|----------|------|
| 40-50 | 可直接用于生产，解决真实痛点 |
| 30-39 | 有明确应用场景，稍加改进可用 |
| 20-29 | 有一定参考价值，但实用性有限 |
| 10-19 | 主要是技术展示，实用性较低 |
| 1-9 | 纯粹娱乐/炫技，无实用价值 |

**加分项**：
- 描述中体现办公/生产力工具属性 +10分
- 描述中明确解决具体问题 +8分
- 描述中体现可复用/可扩展性 +5分
- 描述涉及文档/数据处理类 +5分
- 描述涉及Agent自动化类 +8分

**减分项**：
- 描述中只有展示无实际功能 -10分
- 纯玩具/游戏类（除非教育目的）-8分
- 描述中体现一次性使用场景 -5分

## 输出格式
以 JSON 对象格式输出，不要任何解释：

```json
{
  "interesting_score": 38,
  "useful_score": 10
}
```

- 豆包创意标题：智能3D乐高搭建助手
- 豆包创意描述：只需描述想要的造型，AI即刻生成可交互的3D乐高模型。支持实时预览、零件拆解动画和搭建步骤导出，让创意从想法到立体呈现只需一句话。

**分析**: 标题含"3D"，描述涉及3D编辑器/创作工具，视觉效果好 → 趣味基础分35，创意+5=40；工具类有一定实用性 → 实用22
**输出**:
```json
{
  "interesting_score": 40,
  "useful_score": 22
}
```

**输入**:
- 豆包创意标题：虚拟主播实时驱动系统

**分析**: 标题涉及虚拟形象+实时捕捉，描述中体现3D/实时交互，视觉效果好(趣味高)，VTuber/直播有应用场景(实用中等偏低)
**输出**:
```json
{
  "interesting_score": 38,
  "useful_score": 9
}
```

**输入**:

**分析**: 标题涉及"智能助手"，描述体现Agent自动化类，展示完整自主工作流(趣味中高)，直接可用于生产(实用极高)
**输出**:
```json
{
  "interesting_score": 36,
```
