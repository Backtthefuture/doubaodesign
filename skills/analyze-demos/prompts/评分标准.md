# Demo 评分标准提示词

## 任务
根据 Demo 的 Prompt 描述和预览内容，给出趣味分和实用分两个评分（各50分）。

## 重要原则
1. **趣味分和实用分是独立维度**，高趣味不代表低实用，反之亦然
2. **严格按类型加减分**，不要遗漏加分项
3. **3D/游戏/编辑器类趣味分通常在30-42之间**，除非确实平淡
4. **总分 = 趣味分 + 实用分，满分100分**

## 评分维度

### 趣味分 (1-50)
衡量 Demo 的娱乐性、创意性、视觉冲击力

| 分数区间 | 标准 |
|----------|------|
| 40-50 | 极具创意，视觉震撼，让人惊叹"这也能做到？" |
| 30-39 | 有趣味，展示效果好，能引发兴趣 |
| 20-29 | 一般创意，中规中矩 |
| 10-19 | 缺乏新意，较为平淡 |
| 1-9 | 几乎无趣味性 |

**加分项（必须应用！）**：
- 3D可视化/3D场景 → 基础分30+，再加5-10分
- 游戏类 → 基础分32+，再加5-8分
- 编辑器/创作工具 → 基础分28+，再加5分
- 视觉特效类 → 基础分30+，再加8分
- 操作系统模拟 → 基础分35+
- 视觉效果出众 +5分
- 创意新颖独特 +5分
- "第一次见到这种用法" +8分

**减分项**：
- 纯文本输出 -5分
- 常见功能重复 -5分
- 效果平淡 -3分

### 实用分 (1-50)
衡量 Demo 的实际应用价值、解决真实问题的能力

| 分数区间 | 标准 |
|----------|------|
| 40-50 | 可直接用于生产，解决真实痛点 |
| 30-39 | 有明确应用场景，稍加改进可用 |
| 20-29 | 有一定参考价值，但实用性有限 |
| 10-19 | 主要是技术展示，实用性较低 |
| 1-9 | 纯粹娱乐/炫技，无实用价值 |

**加分项**：
- 办公/生产力工具 +10分
- 解决具体问题 +8分
- 可复用/可扩展 +5分
- 文档/数据处理类 +5分
- Agent自动化类 +8分

**减分项**：
- 纯展示无实际功能 -10分
- 玩具/游戏类（除非教育目的）-8分
- 一次性使用 -5分

|----------|-----------|-----------|---------|------|
| 3D编辑器/创作工具 | 35-42 | 18-28 | 53-70 | 3D乐高编辑器 |

| 物理游戏 | 35-42 | 5-10 | 40-52 | 台球、隧道躲避 |

| Agent自动化 | 30-38 | 38-48 | 68-86 | 飞行追踪、收件箱整理 |
| 文档/图片处理 | 15-25 | 32-42 | 47-67 | 手写食谱解读 |
| 3D可视化/科普 | 32-42 | 8-18 | 40-60 | 宇宙可视化、核电站 |
| 纯图像生成 | 12-22 | 8-18 | 20-40 | 辣椒牛肉面 |

## 输出格式
以 JSON 对象格式输出，不要任何解释：

```json
{
  "interesting_score": 38,
  "useful_score": 10
}
```

## 示例

**输入**: "一次性实现UI、代码构建和各项功能"（3D乐高编辑器）
**分析**: 3D编辑器类，视觉效果好 → 趣味基础分35，创意+5=40；工具类有一定实用性 → 实用22
**输出**:
```json
{
  "interesting_score": 40,
  "useful_score": 22
}
```

**输入**: "读取VRM启动摄像机，面部捕捉+遮罩+遮挡剔除"
**分析**: 3D虚拟形象+实时捕捉，视觉效果好(趣味高)，VTuber/直播有应用场景(实用中等偏低)
**输出**:
```json
{
  "interesting_score": 38,
  "useful_score": 9
}
```

**输入**: "自主规划→编码→浏览器验证"（飞行追踪应用）
**分析**: Agent自动化类，展示完整自主工作流(趣味中高)，直接可用于生产(实用极高)
**输出**:
```json
{
  "interesting_score": 36,
  "useful_score": 46
}
```

**输入**: "手写食谱解读、翻译不同语言手写食谱"
**分析**: 文档处理类，视觉平淡(趣味低)，但解决具体问题(实用高)
**输出**:
```json
{
  "interesting_score": 18,
  "useful_score": 38
}
```
