# 豆包创意内容生成

根据输入的 Demo Prompt 描述，生成**专业且吸引人**的内容。直接输出 JSON，不要任何解释。

## 输出格式（必须严格遵守）
```json
{"title": "6-12字标题", "description": "40-80字描述", "tags": "标签1, 标签2, 标签3", "doubao_tags": "豆包标签1, 豆包标签2"}
```

## 标题风格指南（重要！）

**好的标题特征**：
- **功能+场景明确**：直接说明做什么、用在哪
- **专业但易懂**：体现技术能力但不堆砌术语
- **突出应用价值**：让人一看就知道能解决什么问题
- **避免夸张比喻**：不用"五分钟造个XX"、"浏览器里的XX"等过度营销话术

**标题风格参考**：
| 类型 | 避免（过度营销） | 推荐（专业清晰） |
|------|---------------|---------------|
| 视频理解 | AI秒懂视频玩法 | 智能视频录制助手Sparks |
| 比价工具 | 全网比价神器 | 跨平台智能比价助手 |
| 简历生成 | 简历秒速定制 | 智能简历定制器 |
| 视频分析 | 监控AI火眼金睛 | 监控视频智能分析 |
| 数据可视化 | 图表一键爆炸 | 数据可视化大屏生成器 |
| RPA | 老系统秒变智能 | 企业内网系统RPA助手 |

## 规范
- title: 6-12字，**功能+场景/角色明确**，体现专业性和应用价值
- description: 40-80字，**详细说明完整流程和具体能力**，包含：
  - 使用什么能力（利用豆包XX能力）
  - 做什么事（具体功能点）
  - 产生什么结果（输出物/价值）
- tags: 2-3个标签，体现核心能力特征
- doubao_tags: 1-2个豆包官方能力标签，从以下列表中选择：
  - GUI Agent（图形界面自动化操作）
  - OS Agent（操作系统级自动化）
  - 长视频理解（1280p视频理解能力）
  - 低碰率+高碰率（视频快速浏览+关键帧精看）
  - Video Cut工具（视频剪辑能力）
  - 文档结构化解析（文档智能解析）
  - 复杂指令遵循（多步骤任务执行）
  - 256k上下文（超长文本处理）
  - 多轮对话（长对话管理）
  - 多模态理解（图文视频综合理解）

# 示例Prompt生成

根据Demo的标题和描述，生成一个**具体、可执行**的示例Prompt，模拟真实用户的使用场景。直接输出字符串，不要JSON格式，不要任何解释。

## 规范
- **口语化自然表达**：像真实用户说话一样，不要机器人腔调
- **包含具体场景**：说明用户想做什么、为什么做
- **提供必要细节**：包括具体的输入、参数、期望输出
- **长度控制**：20-50字，简洁但完整

## 示例Prompt风格参考

| Demo类型 | 示例Prompt |
|---------|-----------|
| 企业RPA | 帮我登录公司的OA系统，找到上月的报销申请，导出Excel并发送到我的邮箱 |
| 视频编辑 | 用剪映帮我把这些照片做成一个视频，加上背景音乐和转场效果，然后导出 |
| 开发环境配置 | 帮我在这台Mac上配置React开发环境，安装VSCode、Node.js、Git，并克隆我的项目跑起来 |
| 工作流自动化 | 每天早上帮我自动打开邮箱，检查新邮件，把重要的存到Notion，然后打开日历查看今天安排 |
| 多邻国打卡 | 帮我完成今天的多邻国打卡" |
| 数据搬运 | 帮我整合本月经营数据做报表" |
| 多文档分析 | 这是5份AI项目分析报告和10份行业研究文档上传，帮我综合分析后生成一份项目最佳方案 |
| 长对话管理 | 我们继续昨天没聊完的项目讨论，帮我回顾下我们已经决定了什么，保留关键决策和需求信息 |
| 行程规划 | 帮我规划去日本7天的行程，预算2万，喜欢美食和动漫，帮我查机票酒店，安排每天的行程 |

| 发票归档 | 这些是我出差的发票图片上传多张图，帮我整理成报销单，按日期排序，算出总金额 |
| 商品审核 | 这是待审核的商品图片上传多张，帮我检查有没有违规内容，并与竞品比较价格是否合理 |

## 生成原则
1. **从用户视角出发**：用"帮我..."、"我想..."等第一人称
2. **场景真实可信**：基于Demo的实际功能，不夸大不虚构
3. **包含操作指令**：明确说明要做什么（登录、导出、分析、生成等）
4. **体现价值结果**：说明最终要得到什么（报告、视频、数据等）
