# 豆包Demo评分标准

## 任务
根据豆包Demo的完整描述信息，给出趣味分和实用分（各1-50分）。

**重要**: 每个Demo都有其独特性，必须根据具体内容差异化评分，不能给出相同的分数！

## 输入说明
你将收到以下字段：
- **Demo名称**: 功能名称
- **副标题**: 简短描述
- **使用场景**: 具体应用场景
- **操作步骤**: 实现步骤
- **核心展示**: 核心卖点
- **预期效果**: 期望达成的效果
- **能力分类**: 所属能力类别

## 重要原则
1. **趣味分和实用分是独立维度**，高趣味不代表低实用，反之亦然
2. **必须差异化评分**：相似Demo也要根据细节差异给出不同分数（至少相差2-5分）
3. **严格按类型加减分**，不要遗漏加分项
4. **总分 = 趣味分 + 实用分，满分100分**

## 评分维度

### 趣味分 (1-50)
衡量Demo的娱乐性、创意性、展示效果

| 分数区间 | 标准 |
|----------|------|
| 40-50 | 极具创意，让人惊叹"这也能做到？" |
| 30-39 | 有趣味，展示效果好，能引发兴趣 |
| 20-29 | 一般创意，中规中矩 |
| 10-19 | 缺乏新意，较为平淡 |
| 1-9 | 几乎无趣味性 |

**加分项（必须逐条检查并应用！）**：
- GUI Agent能力（自动操作界面）→ 基础分35+，再加3-8分
- 跨平台/多系统串联 → +5分
- 云手机/移动端Agent → +6分（技术难度高）
- 多步骤自动化流程（3步以上）→ +4分
- 解决痛点的方式新颖独特 → +3-5分
- 视频/图像实时处理 → +5分
- 长文档/长视频处理 → +3分

**减分项**：
- 纯文本对话无交互 → -5分
- 常见功能无创新 → -3分
- 效果平淡无亮点 → -2分

### 实用分 (1-50)
衡量Demo的实际应用价值

| 分数区间 | 标准 |
|----------|------|
| 40-50 | 可直接用于生产，解决真实高频痛点 |
| 30-39 | 有明确应用场景，稍加改进可用 |
| 20-29 | 有一定参考价值，但实用性有限 |
| 10-19 | 主要是技术展示，实用性较低 |
| 1-9 | 纯粹炫技，无实用价值 |

**加分项（必须逐条检查并应用！）**：
- 办公/生产力工具属性 → +8分
- Agent自动化（替代重复劳动）→ +6分
- 解决具体高频问题 → +5分
- 可复用/可扩展到其他场景 → +4分
- 跨平台/多场景适用 → +3分
- 企业级应用场景 → +5分

**减分项**：
- 纯展示无实际功能 → -8分
- 一次性使用场景 → -4分
- 依赖特定环境难以复用 → -3分

## 按能力分类的评分基准

| 能力分类 | 趣味分基准 | 实用分基准 | 说明 |
|----------|-----------|-----------|------|
| 更强Agent能力 | 35-42 | 40-48 | GUI操作、多工具串联加分 |
| 更强多模态能力 | 30-38 | 35-45 | 视频/图像处理加分 |
| 更强LLM能力 | 22-32 | 38-46 | 长文档/推理加分 |

