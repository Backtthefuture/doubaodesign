# 豆包Demo评分标准

## 任务
根据豆包Demo的描述信息，给出趣味分和实用分（各1-50分）。

## 重要原则
1. **趣味分和实用分是独立维度**，高趣味不代表低实用，反之亦然
2. **严格按类型加减分**，不要遗漏加分项
3. **Agent类Demo趣味分通常在30-42之间**
4. **总分 = 趣味分 + 实用分，满分100分**

## 评分维度

### 趣味分 (1-50)
衡量Demo的娱乐性、创意性、展示效果

| 分数区间 | 标准 |
|----------|------|
| 40-50 | 极具创意，让人惊叹"这也能做到？" |
| 30-39 | 有趣味，展示效果好，能引发兴趣 |
| 20-29 | 一般创意，中规中矩 |
| 10-19 | 缺乏新意，较为平淡 |
| 1-9 | 几乎无趣味性 |

**加分项（必须应用！）**：
- Agent自动化类 → 基础分30+，再加5-8分
- 多模态交互类 → 基础分28+，再加5分
- 创意新颖独特 +5分
- 解决实际痛点的方式新颖 +5分
- 多步骤自动化流程展示 +5分

**减分项**：
- 纯文本对话 -5分
- 常见功能重复 -5分
- 效果平淡 -3分

### 实用分 (1-50)
衡量Demo的实际应用价值

| 分数区间 | 标准 |
|----------|------|
| 40-50 | 可直接用于生产，解决真实痛点 |
| 30-39 | 有明确应用场景，稍加改进可用 |
| 20-29 | 有一定参考价值，但实用性有限 |
| 10-19 | 主要是技术展示，实用性较低 |
| 1-9 | 纯粹炫技，无实用价值 |

**加分项**：
- 办公/生产力工具 +10分
- Agent自动化类 +8分
- 解决具体问题 +8分
- 可复用/可扩展 +5分
- 跨平台/多场景适用 +5分

**减分项**：
- 纯展示无实际功能 -10分
- 一次性使用 -5分
- 依赖特定环境 -3分

## 评分参考

| Demo类型 | 趣味分范围 | 实用分范围 | 总分范围 | 示例 |
|----------|-----------|-----------|---------|------|
| Agent自动化 | 32-40 | 38-48 | 70-88 | 智能客服、自动预约 |
| 多模态理解 | 30-38 | 32-42 | 62-80 | 图像分析、视频理解 |
| 文档处理 | 18-28 | 35-45 | 53-73 | 合同审核、报告生成 |
| 对话助手 | 15-25 | 30-40 | 45-65 | 智能问答、知识库 |
| 创意生成 | 28-38 | 20-30 | 48-68 | 文案创作、脚本生成 |

## 输出格式
以JSON格式输出，不要任何解释：

```json
{"interesting_score": 35, "useful_score": 42}
```

## 示例

**输入**: 智能日程助手 - 自动解析会议邮件，创建日程，发送提醒
**分析**: Agent自动化类，解决真实办公痛点 → 趣味基础分32+创意5=37；直接可用于生产 → 实用45
**输出**:
```json
{"interesting_score": 37, "useful_score": 45}
```

**输入**: 多模态文档理解 - 上传图片/PDF自动提取结构化信息
**分析**: 多模态理解类，展示效果好 → 趣味35；文档处理有明确应用场景 → 实用40
**输出**:

